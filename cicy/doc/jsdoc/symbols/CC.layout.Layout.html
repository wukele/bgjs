类 CC.layout.Layout

布局管理器基类
定义在: cicylib-all-debug.js.

类 概述
属性 名称说明
 
CC.layout.Layout()
属性
属性 属性说明
 
ct {CC.Base}
 
ctCS {String}
初始化时添加到容器的样式
 
deffer {Boolean}
延迟多少毫秒后再布局,有利于提高用户体验, 但注意访问同步,例如容器子项在布局时才渲染, 如果deffer已置,则子项渲染将会在JavaScript下一周期调用.
 
invalidate {Boolean} ,默认值:false
指示当引发布局时是否执行布局,如大量引发重复布局的操作可先设置invalidate=true,执行完后再设置invalidate=false,再调用#doLayout布局.
 
itemCS {String}
将子项被加进容器时添加到子项的CSS样式
 
layoutChild
如没有针对单个控件布局的可直接忽略.
 
layoutOnChange
如果每次布局都涉及所有容器子项,则该值应设为true,以便于当容器子项变更(add, remove, display)时重新布局容器
 
wrCS {String}
初始化时添加到容器ct.wrapper的样式
方法
Method Attributes Method Name and Description
 
add(component, cfg)
将组件 component 添加到布局, 如果容器具有有效的布局管理器,则建议从布局管理器添加子项组件, 而不是直接调用容器add方法, 其它子项变更操作(remove, insert)同理.
 
attach(ct)
将布局管理器应用到一个容器控件中。
 
beforeAdd(comp, cfg)
容器在添加子项前被调用。
 
cfgFrom(item)
获得控件当前布局相关的属性配置信息, 该调用无论任何时候都会返回一个非null对象,即使相关配置不存在.
 
detach()
移除容器的布局管理器.
 
display(c, b)
显示/隐藏子项,如果layoutOnChange为true则重新布局容器
 
doLayout()
执行布局,要重写布局应重写onLayout方法.
 
initialize(opt)
初始化布局
 
onLayout()
子项被容器布局后再渲染.
 
remove(comp)
除移子项时调用并重新布局, 注意:从布局管理器移除子控件并不从容器移除.

add(component, cfg)
将组件 component 添加到布局, 如果容器具有有效的布局管理器,则建议从布局管理器添加子项组件, 而不是直接调用容器add方法, 其它子项变更操作(remove, insert)同理. 如果layoutOnChange为true,组件加入后将重新布局容器,否则只调用layoutChild对加入组件进行布局.
